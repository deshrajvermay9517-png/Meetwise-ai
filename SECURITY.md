# Security Policy

We take the security of Meetwise seriously. If you believe you have found a security vulnerability, please read this document to understand how to report it safely and privately.

---

## 🛑 Critical Rules

1. **Do NOT open public issues** (or comment on existing issues) for suspected security vulnerabilities. Doing so exposes the vulnerability to the public before a fix can be prepared.
2. **Do NOT share secrets**, access tokens, Supabase service role keys, API keys (OpenAI/Anthropic/Razorpay/Resend), passwords, or credentials in any public GitHub discussion, issue, or Pull Request.
3. If you accidentally commit a secret, please follow the [GitHub documentation on removing sensitive data](https://docs.github.com/en/code-security/secret-scanning/working-with-secret-scanning-alerts) and rotate/revoke the key immediately.

---

## 🔒 Reporting a Vulnerability

If you identify a security issue, please report it privately:
* **Reporting Channel**: Contact the repository owner [@deshrajvermay9517-png](https://github.com/deshrajvermay9517-png) via private message or the secure email address listed on their GitHub profile.
* **Information to Include**:
  * A detailed description of the vulnerability.
  * Steps to reproduce the issue (including any proofs of concept, code snippets, or sample payloads).
  * The potential impact of the vulnerability if exploited.
  * Any suggestions for fixing the issue.

We will acknowledge receipt of your report within 48 hours and work with you to analyze and patch the issue as quickly as possible.
